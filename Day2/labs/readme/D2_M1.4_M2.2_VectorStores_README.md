# D2 · M1.4 + M2.2 — Vector Stores

**Duration:** ~2.5 hrs across two notebooks · **Type:** Individual, run-and-understand, autograded

## The situation

Before a vector database means anything, you need to see the problem it solves. This module
builds an in-memory semantic search by hand first (M1.4), then stands up two real vector stores —
Chroma and Qdrant — showing the same search working with metadata filtering, and compares them
directly (M2.2).

Both notebooks are fully working code — nothing to write. Run each cell, and read what it shows you.

## Work this side by side with the concept page, not start to finish

- `Day2/concept/D2_M1.4_M2.2_VectorStores_Concept.html` — five concept sections (A1, A2, B, C, D)
- `Day2/labs/starter/D2_M1.4_M2.2_VectorStores_Starter.ipynb` — Part 1 (embeddings + Chroma)
- `Day2/labs/starter/D2_M2.2_VectorStores_Part2_Starter.ipynb` — Part 2 (Qdrant + comparison)

Pattern: read a concept section → run the matching notebook cell(s) → back to the page for the
next concept → repeat, five times across both notebooks. Each notebook's markdown cells tell you
when to switch back to the page.

## Prerequisites

- M1.1, M1.2, M1.3 completed.
- `OPENAI_API_KEY` set as an environment variable.
- `student_config.json` created at the repo root with your real name and GitHub username.
- `chromadb` and `qdrant-client` installed (`pip install chromadb qdrant-client`).
- Docker is **not required** to complete this module — Part 2's Qdrant cell tries a self-hosted
  instance first and automatically falls back to an in-memory client if Docker isn't reachable.
  If your VM has Docker running Qdrant (`docker run -p 6333:6333 qdrant/qdrant`), you'll get the
  production-path stretch credit; if not, the fallback still passes the graded checks.

## Part 1 — Embeddings & Chroma

1. Open the concept page and Part 1's notebook side by side.
2. Run the **Setup** cell first.
3. **Concept A1 → Tasks 1-2.** Read Concept A1 (rank-by-meaning demo), then run the cell that
   embeds 3 warm-up sentences and indexes all 10 real Meridian passages.
4. **Concept A2 → Tasks 4-5.** Read Concept A2 (keyword vs. semantic demo), then run the cell
   comparing both search methods and evaluating all 5 fixed queries — **this is M1.4's graded
   deliverable.**
5. **Concept B → Chroma.** Read Concept B (scan-time and metadata-filter demos), then run the
   cell that stands up a real Chroma collection using the exact same embeddings from Task 2, and
   runs a filtered vs. unfiltered query.
6. Run the final cell — it saves `m1_4_m2_2_part1_results.json` next to the notebook.

## Part 2 — Qdrant & Choosing a Store

Part 2's notebook is self-contained — it re-embeds the same 10 passages, so it works even if
Part 1's kernel is gone.

1. **Concept C → Qdrant.** Read Concept C (Docker vs. in-memory toggle), then run Setup and the
   Qdrant cell. Read which connection mode printed for you.
2. **Concept D → Comparison.** Read Concept D (choosing a store), then run the cell that stands
   up Chroma again alongside Qdrant and runs the identical query against both.
3. Run the final cell — it saves `m2_2_part2_results.json` next to the notebook. **This is what
   makes M2.2 gradable** — Part 1's Chroma alone wasn't the full deliverable.
