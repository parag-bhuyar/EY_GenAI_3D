## The situation

Yesterday's M1.2 covered how to turn a messy customer message into clean, structured data — name, account type, intent, urgency, a one-sentence summary. The lab that would have built it didn't happen. Today's notebook shows it built for real, using LangChain — the framework the rest of this programme runs on. Same job M1.2 described. Different, more modern way of doing it.

This notebook is fully working code — nothing to write. Run each cell, and read what it shows you.

## Work this side by side with the concept page, not start to finish

This module has two files, meant to be used together, not one after the other:

- `Day2/concept/D2_M2.1_LangChain_Core_Concept.html` — four short concept sections (A, B, C, D)
- `Day2/labs/starter/D2_M2.1_LangChain_Core_Starter.ipynb` — the notebook below

The pattern: read Concept A on the page → run the matching cell(s) in the notebook → come back

## How to work through this module

1. Open `Day2/concept/D2_M2.1_LangChain_Core_Concept.html` in your browser and
   `Day2/labs/starter/D2_M2.1_LangChain_Core_Starter.ipynb` in Jupyter or VS Code — side by side if
   your screen allows it.
2. Run the notebook's **Setup** cell first.
3. **Concept A → Task 1.** Read Concept A (the provider-swap toggle), then run Task 1's cell:
   `init_chat_model` — one interface, any provider.
4. **Concept B → Tasks 2–4.** Read Concept B (click through the four chain stages), then run the
   cell that builds the prompt template, the extraction chain, and the retry wrapper, and tests
   it on one message.
5. **Concept C → Task 5.** Read Concept C (the manual-vs-chain line-count toggle), then run the
   cell that runs the chain against all 10 sample messages — this is the graded deliverable.
6. **Concept D → Task 6.** Read Concept D (the agent decision demo), then run the read-along cell
   showing a real minimal tool-using agent. Full hands-on agent-building is Day 3's M3.1.
7. Read the Recap section on the page, then run the notebook's final cell — it saves
   `m2_1_extraction_results.json` next to the notebook. Without that file, the autograder has
   nothing to check.

## Running the autograder

```
python Day2/autograder/D2_M2.1_LangChain_Core_Autograder.py
```

Checks: no deprecated LangChain APIs anywhere in the notebook, and at least 8 of the 10 sample

the notebook itself). On pass, push the result:

```
git push origin main
```
