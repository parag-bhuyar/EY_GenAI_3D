# D1 ¬∑ M1.2 ‚Äî Building on the OpenAI API

**Duration:** 2.0 hrs ¬∑ **Type:** Individual, autograded ¬∑ **Capstone:** this lab's output is your Capstone Phase 1 extraction module

## Objective

Build a structured-extraction module that turns a free-text Meridian Retail Bank customer message
into a validated JSON record, using tool calling ‚Äî not string parsing, not regex.

## Prerequisites

- M1.1 completed (you should already be comfortable calling the Chat Completions API).
- `OPENAI_API_KEY` set as an environment variable.
- `pydantic` and `openai` installed (already on the lab VM image; if working locally, `pip install
  pydantic openai`).

## Setup

1. Open `Day1/labs/starter/D1_M1.2_OpenAI_API_Starter.py`.
2. Read the `CustomerRequest` schema at the top ‚Äî this is the contract your extraction must
   satisfy. Don't change its fields; the autograder depends on this exact shape.
3. There are two TODOs: the tool's `parameters` JSON schema, and the body of
   `extract_customer_request`.

## Steps

1. **Fill in `EXTRACTION_TOOL["function"]["parameters"]`** ‚Äî a JSON-schema object describing
   `name`, `account_type`, `intent`, `urgency`, and `customer_summary`, matching `CustomerRequest`.
   Use `type: "object"`, `enum` lists for the constrained fields, and a `required` list.
2. **Implement `extract_customer_request`**:
   - Call `client.chat.completions.create(...)` with `tools=[EXTRACTION_TOOL]` and `tool_choice`
     forcing that specific tool to be called.
   - Parse the tool call's `arguments` JSON string.
   - Validate it against `CustomerRequest(**parsed_args)`.
   - On a validation or parsing failure, retry (up to `max_retries`) instead of crashing ‚Äî this is
     the "robustness" half of this module's topics, and the autograder specifically checks that
     your code recovers from one simulated bad response before it checks the 10 sample messages.
3. Test manually: `python Day1/labs/starter/D1_M1.2_OpenAI_API_Starter.py` should print a valid
   JSON record for the sample message at the bottom of the file.

## Running the autograder

```
python Day1/autograder/D1_M1.2_OpenAI_API_Autograder.py
```

This runs your `extract_customer_request` against 10 real sample customer messages
(`Day1/data/D1_M1.2_sample_customer_messages.json`) ‚Äî it makes 10 real API calls, so it costs a
small amount each run; don't run it in a loop while debugging, test manually against one message
first (Step 3). You need at least 8/10 to pass. On pass, push the result:

```
git add Day1/submissions/D1_M1.2.json
git commit -m "D1 M1.2 submission"
git push origin main
```

Extracting all 10/10 correctly counts as this module's stretch/Diamond signal ‚Äî no separate
stretch task to build.
