# D1 · M1.1 — LLM Mechanics for Practitioners

**Duration:** 1.5 hrs · **Type:** Individual, autograded

## Objective

Build an annotated notebook comparing how the OpenAI Chat Completions API behaves under different
decoding settings — temperature, top_p, max_tokens, stop sequences — and how two models trade off
latency and token usage on the same Meridian Retail Bank support scenario.

## Prerequisites

- Completed the pre-joining self-paced Python/GenAI courses.
- `OPENAI_API_KEY` set as an environment variable on your lab VM (do not hard-code it anywhere).
- `student_config.json` created at the repo root from `student_config.example.json`, with your
  real name and GitHub username filled in — every module's autograder needs this.

## Setup

1. Open `Day1/labs/starter/D1_M1.1_LLM_Mechanics_Starter.ipynb` in Jupyter or VS Code.
2. Confirm the first code cell runs without an assertion error (that's your API key check).
3. Work through Tasks 1–4 in order — Task 3 reuses the same prompt from Task 1, so don't skip
   ahead.

## Steps

1. **Task 1 — Temperature sweep.** Call the model at temperature 0, 0.7, and 1.0, twice each, on
   the provided banking prompt. Record latency and token usage for every call.
2. **Task 2 — Sampling controls.** Run the four variants described in the notebook (`top_p=0.1`,
   `top_p=1.0`, `max_tokens=20`, `stop=["."]`) and record each one.
3. **Task 3 — Model comparison.** Call both models named in the setup cell on the same prompt and
   compare latency/tokens. Do not hard-code a dollar price — check the current pricing page
   yourself if you want to reason about cost.
4. **Task 4 — Annotate.** Replace the placeholder markdown cell with your own genuine reflection —
   this is graded on substance, not length, but it does need to reference your actual numbers.
5. **Task 5 — Stretch (optional, Diamond tier).** Quantify reproducibility across 5 runs at each of
   temperature 0 and 1.0.
6. Run the final cell — it saves `m1_1_comparison_results.json` next to the notebook. Without this
   file, the autograder has nothing to check.
